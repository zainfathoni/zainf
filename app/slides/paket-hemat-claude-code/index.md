# 1. Membangun kesadaran bersama AI

---

## The 70% Problem

- AI tools sangat bagus untuk menyelesaikan ~70% masalah
- 30% sisanya memerlukan keahlian manusia dan critical thinking
- Pahami batasan AI untuk ekspektasi yang realistis

::: notes
AI excels at common patterns but struggles with edge cases and domain-specific nuances.
:::

---

# 2. Cara minimalisir halu AI

---

## Context Engineering

Bringing engineering principles to AI context management

- Clear, specific prompts
- Structured input/output formats  
- Iterative refinement
- Document your context patterns

---

## Making tech stack AI-friendly

Design your architecture with AI collaboration in mind

- Clear naming conventions
- Comprehensive documentation
- Modular, testable code structure
- Consistent patterns across codebase

---

## When AI goes wrong

Recognizing and handling AI mistakes

- Code reviews are essential
- Test thoroughly
- Don't blindly trust AI output
- Have rollback strategies ready

---

## Trust, but Verify

A systematic approach to AI-assisted development

- Use AI for initial implementation
- Always verify the output
- Test edge cases manually
- Code review with human expertise

---

## Automated testing

Your safety net for AI-generated code

- Unit tests for core logic
- Integration tests for workflows
- End-to-end tests for user journeys
- Regression tests for AI changes

# 4. Tips biar efektif nan hemat dengan Claude Code

---

## Output tokens are expensive

**ALWAYS use "Plan mode"**

- Think before you code
- Reduce unnecessary iterations
- Get approval before implementation
- Batch related changes together

---

## Similar to Kiro's Specs concepts

Structured approach to development

- Clear specifications
- Defined requirements
- Measurable outcomes
- Documented decision making

[kiro.dev/docs/specs/concepts](https://kiro.dev/docs/specs/concepts/)

---

## Best Practices Summary

1. **Plan first** - Use plan mode to save tokens
2. **Context is king** - Provide clear, structured context
3. **Test everything** - AI-generated code needs verification
4. **Iterate wisely** - Make targeted, specific requests
5. **Document patterns** - Build reusable prompt libraries
